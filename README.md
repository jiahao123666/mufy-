# mufy-新手正则入门
 **Mufy AI 对话渲染正则零基础入门教程**

## 什么是正则？


*正则表达式 Regular Expression，简称正则，是一套用来匹配、查找、替换文本规则的语法。
简单说：给电脑写一套“文字筛选模板”，快速从一堆文字里找出符合格式的内容*


## 什么是mufy?

*Mufy AI 是一款移动端AI虚拟角色对话平台（安卓/iOS都有），主打高自由度角色扮演、剧情互动，正则,HTML/CSS卡片渲染是它的高阶玩法，本仓库主要介绍有关于mufy正则的入门内容
如何使用正则？（在通用模式下）*


## 一、正则表达式

*根据基础的正则知识，mufy的正则可以被写为
```xml
<变量>([\s\S]*?)</变量>
```
就比如说
```xml
<小手机>([\s\S]*?)</小手机>
```
如果你想要更高阶的正则，可以询问AI使用更高阶的正则，不过要注意的是，mufy本身只是一个简单的Ai聊天软件，对于正则而不会更加包容的设定，因此在可能会导致渲染失败*


## 二、替换字符串

**替换字符串类就是我们所需要的代码，浏览器通过html/css进行渲染,达到我们的目的，如果你使用 AI,那么把AU给你的代码放入其中便可,然而，这时常会出现一些问题，无论是官方的指导文档，还是 AI生成的代码,虽然在生成过程中有```作为关键,然而却会被浏览器或是应用直接过滤掉,然而你如果不是连续的代码的话,那么```是必不可少的，否则就会出现明明代码，正确可是无法渲染的问题，我们可以使用例如deepseek这一种能够复制全部文本,不忽略渲染文本的AI,同时我也发布了一些我个人的正则在Releases,对于替代字符串方面，不建议直接套用官方的替代字符串，否则很大概率渲染失败**


## 三、测试

*在修改正则的页面点击右上角的测试就可以测试了，注意是修改正则的那一个页面，而不是创建正则的那个页面*
*如果测试完成，就可以开始正常对话了，想要让渲染的文本出现，只需要在 AI说完它该说的话之后点击“修改”，将正则表达释放上去（也就是短短的那一串，不是你的全部代码，因为此时你已经用正则表达式代替代码了）就可以实现渲染效果了*


## 四、测试文本

*我们可以输入测试文本进行测试，比如说小手机的xml
```xml
<小手机>
  <状态>
    <电量>85</电量>
    <时间>14:30</时间>
    <信号>3</信号>
  </状态>
</小手机>
```
请注意,因为渲染问题xml被挤掉了,真正的文本前面还要再加上xml才能正常使用,如果不行，那就再次在文本前后加入```
## 五、特殊例子

*当我们在渲染两个页面的时候，就可以选择适当抛弃一些东西，比如说前文的```可以选择去除，xml也是一样,不过去除也有情况！如果你能够正常渲染图层，然而在你渲染的图层的前面或后面有“/\html><$1?xml”等东西,可能阻碍视觉，那么此时你就可以删了,如果没有请别删，不过删了之后可能会造成运行有问题，所以需要多加注意，谨慎行事*


## 六、一些好玩的例子

*在你的html能够正常渲染之后，你可以在设置里面开通浏览器存储权限（安卓13以上只需要开通图片上传或文本上传就可以，实在不行再开存储权限）然后让aa给你修补好一个能够上传图片或是文件的按钮，接着你就可以给你的AI发送图片或是文件了,你可以试着发送一个html用作 App，这样子就能完美解决在一个正则里面继续渲染别的正则（其实是换种方式）你也可以选择，先把你要渲染的正则app放在前面，手机正则放在后面，比如*
```xml
<gmail>([\s\S]*?)</gmail>
<小手机>
  <状态>
    <电量>85</电量>
    <时间>14:30</时间>
    <信号>3</信号>

```
## 就可以同时实现渲染两个正则的效果！不过如果你可以用
<变量1>动态变量</变量1>
<变量2>动态变量</变量2>
的格式，建议你这么使用，这样成功率更高。
